# WeOpen 红熊猫吉祥物 Redy 完整视觉方案

本方案整合了 WeOpen 红熊猫吉祥物 Redy 的核心创意、视觉规范、应用场景及全量扩展内容，融合吉卜力温润手绘风格与极客开源文化，涵盖人设、视觉、落地应用、动态效果等所有维度，可直接交付设计师、厂家及运营人员执行，确保全场景风格统一、细节精准。

# 一、方案总览与吉祥物档案

## 1\.1 方案核心定位

吉祥物：Redy（命名逻辑：Red \+ Ready）

核心风格：吉卜力温润手绘质感（柔边线条、低饱和色彩、漫反射光影）× 极客审美（简洁模块化、Git 文化、像素细节）

核心定位：WeOpen 开源社区守护者、代码园丁、分支探险家，连接温润的技术氛围与严谨的极客精神。

## 1\.2 吉祥物完整人设

- 基本信息：物种为红熊猫，性格冷静、极客、专注、温和可靠，乐于 Pick 优质代码与优秀贡献者。

- 口头禅：Ready to merge\. / Let’s pick the good stuff\. / Keep it open, keep it warm\.

- 核心符号寓意：红熊猫（小巧敏捷、专注无害）；飞行护目镜（深入代码底层、调试视角）；工具腰带（开源工具箱、项目生态）；Git 分支树（开源森林、协作生长）。

- 标志性特征：复古飞行护目镜（镜片带 Git 分支绿光）、工具腰带（Banana AI 终端\+Grape Git 分支挂件）、蓬松大尾巴（像素笔刷质感）。

## 1\.3 对外品牌文案

- 主 Slogan：WeOpen · Redy \| Open source, warm and ready\.

- 角色介绍（短版）：Redy 是 WeOpen 的开源守护者，一只温润又极客的红熊猫。它穿梭在代码森林里，细心 cherry\-pick 每一份优秀贡献，让开源更温暖、更可靠、更 Ready。

- 项目理念句：温和看待技术，严谨对待代码；开源不是冰冷的指令，是一群人一起生长的森林；Ready to open, ready to collaborate\.

# 二、视觉设计规范（设计师可直接执行）

## 2\.1 艺术风格指南

### （1）主格调：吉卜力温润感

- 线条：带有手绘质感的棕褐色柔边线条，无硬锐黑色线条，轻微手绘抖动感。

- 色彩：降低饱和度，固定色值如下：主体锈红\#C14B30、奶油白\#F6F1E9、深炭灰\#2D2A2E、Git 绿\#2CC68E、暖米背景\#F8F5F0、柔边棕褐\#8B6E54。

- 光影：采用温和的漫反射，边缘添加淡淡的暖色光晕（浅橙黄），营造温暖“工作室”氛围，无强烈对比阴影。

- 质感：水彩薄涂 \+ 轻微毛发肌理，突出红熊猫蓬松柔软的特质。

### （2）辅助风格：极客像素感

- 护目镜镜片：8\-bit 像素反光，可显示 \&gt;\_ 终端光标，带有 Git 分支绿光效果。

- 细节点缀：工具腰带边缘、尾巴末端可添加轻微像素纹理，避免过度赛博，保持温暖基调。

## 2\.2 标准造型库（全场景适配）

- 造型 A：极简头像/Logo 图标（1:1 方形）—— 红熊猫侧脸轮廓，护目镜为核心识别点，圆形负空间构图，可缩至 16×16px favicon，单色/双色版本适配徽章、贴纸。

- 造型 B：标准全身立绘（4:5 比例）—— 蹲坐姿态，护目镜拉下，尾巴自然圈绕，腰带挂件清晰，适用于官网、GitHub 头图、海报。

- 造型 C：园丁形态（16:9 Banner 级）—— 坐在 Git 分支构成的树枝上，手持放大镜挑选“水果项目”，背景为吉卜力风森林\+代码树叶\+像素果实，寓意开源森林培育。

- 造型 D：Q 版迷你形态（1:1 比例）—— 大头小身，适用于 404 页面、加载动画、提示弹窗、贴纸，动作包括歪头、挥手、抱终端等。

# 三、Figma 组件规范（设计师直接套用）

## 3\.1 核心原则

组件可复用、可缩放，保持视觉一致性，适配不同场景（网页、海报、周边、图标），所有尺寸、色值、间距固定，无需二次调整。

## 3\.2 基础色板（Figma 色值直接复制）

|颜色名称|色值（Hex）|用途|备注|
|---|---|---|---|
|主体锈红|\#C14B30|Redy 主体毛发、主视觉元素|低饱和度，无渐变，可添加轻微纹理|
|奶油白|\#F6F1E9|腹部、胸口、耳尖、背景|柔和不刺眼，作为浅色搭配|
|深炭灰|\#2D2A2E|工具腰带、挂件细节、文字|替代纯黑，更温润|
|Git 绿|\#2CC68E|护目镜微光、Git 符号、强调色|高亮但不突兀，作为极客元素点缀|
|暖米背景|\#F8F5F0|页面背景、海报背景|与奶油白呼应，营造温暖氛围|
|柔边棕褐|\#8B6E54|线条、轮廓|柔边，不生硬，宽度1\-2px|

## 3\.3 组件尺寸规范

### （1）核心组件（基础可复用）

- 标准立绘组件：800×1000px（比例4:5），可等比缩放，最小缩放至200×250px（不失真）

- Logo/头像组件：
        \- 圆形版：512×512px（适配头像、徽章）
        \- 极简图标版：128×128px（可缩至16×16px favicon）
      

- Q版迷你组件：400×400px（比例1:1），适配弹窗、贴纸、404页面

- 场景插画组件（园丁场景）：1920×1080px（Banner比例16:9），可裁剪适配不同Banner尺寸

### （2）间距与圆角规范

- 组件间距：所有Redy组件与文字、其他元素的间距≥24px（网页）、≥16px（贴纸、徽章）

- 圆角规范：
        \- 圆形组件：圆角100%（正圆形）
        \- 方形组件：圆角16px（柔和边角，贴合吉卜力风格）
        \- 贴纸组件：圆角24px（更圆润，适配物料）
      

### （3）组件层级（Figma图层顺序）

1. 底层：背景层（暖米\#F8F5F0，无纹理）

2. 中层：Redy主体层（毛发、身体，添加轻微水彩纹理）

3. 上层1：细节层（护目镜、工具腰带、挂件，护目镜添加微光效果）

4. 上层2：光影层（柔光、光晕，不叠加，透明度30%）

5. 顶层：极客点缀层（像素纹理、Git符号，透明度20%，不突兀）

# 八、动态效果描述（网页/运营/视频适配）

核心原则：动态柔和，不夸张，贴合Redy冷静极客\+温润的气质，突出极客细节，适配网页、加载动画、社群表情等场景，可直接给动效设计师参考。
