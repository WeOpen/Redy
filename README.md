# WeOpen 吉祥物 Redy 正式设计方案

**项目名称：** WeOpen 品牌吉祥物与视觉识别系统设计
**吉祥物名称：** Redy
**适用范围：** 品牌传播 / 官网与 GitHub / 社区运营 / CLI 工具 / 周边物料 / 动效与表情体系

---

# 1. 项目背景与设计目标

## 1.1 项目背景

WeOpen 作为一个具有开源属性与技术社区气质的组织，需要一套具有长期识别性、扩展性与情感温度的品牌视觉符号，用于统一组织形象、强化社区记忆点，并在 GitHub、官网、CLI、社群运营和周边物料等多个场景中形成稳定输出。

在视觉表达上，WeOpen 希望兼具两种气质：
一是温润、柔和、有陪伴感的手绘氛围；
二是简洁、模块化、具有 Git 与开源文化隐喻的极客审美。

因此，本方案以红熊猫吉祥物 **Redy** 为核心，构建一套兼具品牌人格、视觉规范、交互语言与衍生能力的完整系统。

## 1.2 设计目标

本项目的核心目标包括：

1. 建立 WeOpen 可长期使用的品牌吉祥物形象；
2. 统一组织在不同媒介中的视觉识别；
3. 为设计、开发、运营和物料制作提供标准化执行依据；
4. 让吉祥物不仅是“插画形象”，更成为具有品牌叙事能力的“生态角色”；
5. 在温暖亲和与技术专业之间建立清晰平衡。

# 2. 吉祥物品牌定位

## 2.1 吉祥物命名

**Redy**
命名由 **Red + Ready** 组合而来，既对应红熊猫的核心色彩，也传达 “准备就绪、可靠、可协作” 的品牌气质。

## 2.2 核心角色定位

Redy 不只是 WeOpen 的卡通形象，而是组织的品牌人格化代表。其正式定位为：

**WeOpen 开源生态维护者 / 代码园丁 / 分支探险家**

这一定位用于统一所有视觉与文案解释体系：

* **红熊猫**：敏捷、专注、友好、温和而可靠
* **护目镜**：深入代码细节、调试与观察视角
* **工具腰带**：技术工具箱与项目生态能力
* **Git 分支树 / 开源森林**：协作式生长、版本演化、开源共建
* **Cherry-pick 行为**：精选优质贡献、组织有价值的代码成果

## 2.3 品牌人格关键词

Redy 的品牌人格建议统一使用以下关键词描述：

* 冷静
* 极客
* 温和
* 可靠
* 专注
* 审慎
* 观察者
* 维护者
* 共建者

## 2.4 品牌语义主张

围绕 Redy，可统一形成 WeOpen 的品牌表达：

* 技术可以是温暖的
* 协作不是冷冰冰的流程，而是共同生长的生态
* 开源不是无序堆叠，而是被精心维护的森林
* WeOpen 不只是构建工具，也在挑选、整理与培育更好的开源体验

# 4. 视觉概念与风格策略

## 4.1 总体艺术方向

本方案采用双重融合风格：

### 4.1.1 主风格：温润手绘感

参考方向为具有工作室氛围的手绘插画风格，强调：

* 柔边棕褐线条
* 低饱和暖色系
* 漫反射光影
* 水彩薄涂与毛发肌理
* 温和、安静、具有陪伴感的画面氛围

### 4.1.2 辅风格：极客像素感

作为辅助层嵌入，不可喧宾夺主。主要用于：

* 护目镜反光
* 工具挂件
* Git 分支符号
* 终端光标元素
* 尾巴与配件局部像素纹理

整体原则是：
**主体温润，细节极客；整体柔和，局部精确。**

# 5. Redy 角色设计规范

## 5.1 基本设定

* **物种：** 红熊猫
* **体态：** 圆润、轻盈、蓬松、敏捷
* **表情基调：** 冷静、专注、略带观察感，不做过度夸张的卡通表情
* **动作气质：** 克制、安静、有判断力，不做喧闹型 mascot

## 5.2 标志性特征

### 护目镜

护目镜是 Redy 的第一识别点，必须在多数版本中保留。
要求如下：

* 复古飞行护目镜造型
* 镜框偏浅灰或炭灰
* 镜片可加入轻微 Git 绿发光
* 可在镜片中嵌入终端光标 `>_` 或 Git 分支反光细节
* 既有“观察”感，也有“技术工具”感

### 工具腰带

腰带用于表达 Redy 的“维护者 / 园丁 / 工具型角色”身份。

标准挂件配置：

* **Banana AI 终端**：象征 AI 工具、模块能力、项目工具链
* **Grape Git 分支挂件**：象征版本演化、分支协作与开源文化

### 尾巴

尾巴是 Redy 的第二识别特征。
要求蓬松、层次感明显，并可承担更多图形抽象能力：

* 在角色本体中表现毛发柔软感
* 在系统图形中抽象为“像素笔刷”或动态拖尾
* 在动效中承担情绪反馈和状态反馈作用

# 6. 标准视觉规范

## 6.1 标准色板

以下颜色建议作为官方统一品牌色板使用：

| 颜色名称  |      色值 | 用途                |
| ----- | ------: | ----------------- |
| 主体锈红  | #C14B30 | Redy 主体毛发、主视觉主色   |
| 奶油白   | #F6F1E9 | 胸口、腹部、耳尖、浅色背景     |
| 深炭灰   | #2D2A2E | 腰带、挂件、文字、结构细节     |
| Git 绿 | #2CC68E | 护目镜微光、Git 符号、强调反馈 |
| 暖米背景  | #F8F5F0 | 网页与海报主背景          |
| 柔边棕褐  | #8B6E54 | 轮廓线条、结构边线         |

## 6.2 用色原则

1. 主视觉色彩以锈红、奶油白、暖米为主体；
2. 深炭灰替代纯黑，避免气质过冷；
3. Git 绿只能作为强调色，面积不宜过大；
4. 禁止使用高饱和荧光色破坏整体温润基调；
5. 允许在大幅场景插画中使用柔和暖光渐层，但主体角色不建议使用强烈赛博渐变。

## 6.3 线条与质感规范

* 轮廓采用柔边棕褐，不使用纯黑硬边
* 线条允许轻微手绘抖动感
* 毛发表现采用薄水彩叠加或浅纹理刷
* 光影以漫反射和暖色边缘柔光为主
* 避免高反差、高金属质感和过度赛博化效果

# 7. 标准造型系统

## 7.1 造型 A：Logo / 极简头像

**用途：** favicon、社群头像、徽章、组织图标、小尺寸识别

设计要求：

* 采用侧脸或探头构图
* 护目镜为识别核心
* 可在 16×16px 下保持辨识
* 需同时输出单色版、双色版和反白版

## 7.2 造型 B：标准全身立绘

**用途：** 官网、海报、GitHub、运营主视觉

设计要求：

* 蹲坐或轻前倾观察姿态
* 护目镜、尾巴、腰带挂件齐全
* 保持角色稳定、可复用、可换姿态扩展

## 7.3 造型 C：园丁场景版

**用途：** Banner、开源社区主视觉、活动页头图

设计要求：

* Redy 坐在 Git 分支树枝上
* 手持放大镜进行 cherry-pick
* 周围悬挂项目水果元素（Orange / Banana / Grape）
* 背景为“开源森林”概念图景

## 7.4 造型 D：Q 版迷你版

**用途：** 404 页、加载动画、弹窗提示、社群贴纸

设计要求：

* 大头小身
* 动作简单明确
* 适合情绪化表达与小尺寸场景传播

# 8. 三视图描述规范

为保障不同画师和供应商执行统一，Redy 需具备标准三视图描述。

## 8.1 正面视图

* 正蹲坐，身体整体对称
* 头部圆润，耳朵外轮廓明显
* 护目镜居中覆盖眼部区域
* 腹部与胸口奶油白形成视觉中心
* 工具腰带在正面可见，挂件左右分布明确
* 尾巴在身体后方展开，与身体宽度基本相当

## 8.2 侧面视图

* 身体轻微前倾，体现观察与专注
* 鼻尖、耳朵与护目镜轮廓清晰
* 腰带与挂件有前后层次
* 尾巴自然弯曲，贴合身体侧方
* 护目镜高光与 Git 绿反光保留

## 8.3 背面视图

* 背部圆润，毛发方向自然向后延展
* 护目镜绑带从后脑绕过
* 工具腰带背面保留基础搭扣结构
* 尾巴蓬松扇形展开，可带轻微奶油白尾端细节
* 背面不出现多余复杂装饰，以便供应商简化制作

# 10. Figma 组件与设计资产规范

## 10.1 设计原则

* 可复用
* 可缩放
* 多场景适配
* 保持气质统一
* 降低后续二次修改成本

## 10.2 推荐基础资产尺寸

| 组件   |         建议尺寸 | 用途           |
| ---- | -----------: | ------------ |
| 标准立绘 |  800×1000 px | 海报、官网、主视觉    |

* 网页中 Redy 与文字最小安全距离建议不小于 24px
* 小尺寸贴纸与徽章应用中最小安全距离不小于 16px
* 图标版不建议加入多余背景装饰，留白优先

## 10.4 图层建议

1. 背景层
2. 主体层
3. 细节层
4. 光影层
5. 极客点缀层

极客点缀层透明度不宜过高，避免破坏整体手绘氛围。

| 场景  | Redy 行为      | 含义         |

这一系统可直接映射到 UI、CLI 和运营插画。

# 12. 动效设计规范

## 12.1 总体原则

Redy 的动效应保持“柔和、克制、可爱但不闹腾”的节奏。
不建议使用强烈弹跳、夸张拉伸、过多粒子喷发等偏游戏化表现。

## 12.2 护目镜动效

* 常态：保留弱反光与微弱 Git 绿点
* Hover：镜片绿光缓慢流动，时长约 1.5 秒
* 成功反馈：闪烁 3 次后恢复常态

## 12.3 尾巴动效

* 待机：轻微缓慢左右摆动
* 开心：摆动幅度增加
* 困惑：尾巴下垂，停止摆动
* 专注：整体静止，仅尾尖微颤

## 12.4 眼神跟随

适用于官网或 Banner：

* 幅度小
* 延迟柔和
* 不做夸张追踪
* 保留“观察者”人设气质

## 12.5 Loading 动效

建议使用 Q 版 Redy：

* 尾巴绕体缓慢转动
* 镜片内部 Git 绿线流动
* 身体轻微浮动
* 加载完成后镜片闪一下

# 13. 应用场景规范

## 13.1 组织 Logo

Logo 采用极简侧脸结构，Redy 从圆形负空间中探出头部。
适合 favicon、GitHub organization icon、徽章与小程序入口等高频识别场景。

## 13.2 GitHub Header / Banner

推荐标准布局：

* 左侧：Redy 半身或园丁形态
* 中部：Git 分支结构延展
* 右侧：WeOpen 标志文字与 slogan

建议标语：
**Build. Pick. Evolve.**
或
**Open source, warm and ready.**

## 13.3 项目协同主视觉

采用“开源森林 / 园丁概念”：

* Redy 在 Git 分支树上 cherry-pick
* 项目成果以水果隐喻呈现
* 代码块与像素叶片构成场景语言

此方案适合：

* 社区介绍页
* 开源活动 KV
* 贡献指南封面
* 年度总结海报

## 13.4 404 / 异常页面

推荐使用 Q 版困惑 Redy：

* 低头寻找
* 手持放大镜
* 背景轻量，不抢文案信息

* **Branch not found.**
* **Redy is searching the open forest...**
* **Looks like this path hasn’t been merged yet.**

## 13.5 CLI 欢迎界面

建议保留 Redy 的极客仪式感。正式推荐文案如下：

## 13.6 更新日志 / 贡献指南

建议为每类更新固定插画行为：

* Merge 完成：Redy 点头

# 16. AI 绘图与草图执行说明

## 16.1 AI 绘图用途

AI 提示词适合用于：

* 概念草图探索
* 构图方向验证
