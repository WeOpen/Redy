# 悬浮页脚按钮设计

## 背景

当前页脚操作入口位于左下角，采用文本链接形式，视觉权重偏弱，也不利于在长页面中持续提供快速操作入口。用户希望将“回到顶部”和“Explore the repository”改成图标按钮，并以悬浮方式固定在页面右下角。

## 目标

1. 将两个文本链接改为图标按钮
2. 按钮组固定在页面右下角，滚动时始终可见
3. 保持当前暖色、轻磨砂、卡片式视觉风格
4. 不引入额外图标库，仅做最小实现

## 方案选择

### 方案一：极简线框横向按钮

优点是与现有布局融合自然，但不符合用户想要的“悬浮固定在右下角”。

### 方案二：悬浮固定竖向按钮组（最终选择）

- 使用 `position: fixed`
- 两个按钮上下排列
- 上方按钮为回到顶部
- 下方按钮为 GitHub 仓库入口

优点是功能清晰、长页面可达性更好、实现简单。

### 方案三：纯图标无边框悬浮

视觉更轻，但点击区域与可发现性较弱，因此不采用。

## 最终设计

### 1. 结构

- 在首页组件中保留页脚文案
- 移除页脚原有文本链接
- 新增一个独立的悬浮按钮组容器
- 悬浮按钮组包含两个按钮：
  - 回到顶部
  - 打开 GitHub 仓库

### 2. 视觉

- 按钮组固定在页面右下角
- 两个按钮竖向排列
- 使用圆角矩形或圆角方形按钮
- 延续页面现有暖色调、半透明卡片感、细边框与轻阴影
- 使用内联 SVG 图标，不引入第三方图标依赖

### 3. 交互

- 回到顶部按钮链接到 `#hero`
- GitHub 按钮保留外链行为
- hover / focus 状态增强边框、背景或阴影反馈
- 保留 `aria-label` 以保证无障碍

### 4. 响应式

- 桌面端显示为右下角固定悬浮按钮组
- 移动端保留固定定位，但适当缩小尺寸和间距
- 避免遮挡主要文案与核心 CTA

## 验收标准

- 页面右下角有固定悬浮按钮组
- 包含“回到顶部”和“GitHub 仓库”两个图标按钮
- 两个按钮功能正常
- 页脚左下角不再保留重复文本链接
- 页面预览无明显回归，无控制台错误

## 原则应用

- **KISS**：只改首页结构和样式，不动主体内容
- **YAGNI**：不引入图标库，不做额外动画系统
- **DRY**：移除重复文本入口，统一成悬浮按钮组
- **SOLID**：将操作入口从页脚文案区分离，职责更清晰
